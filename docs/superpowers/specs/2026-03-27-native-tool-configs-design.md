# Native Multi-Tool Config Generation

## Summary

Extend `yuva init` to generate rich, native configuration files for each AI coding tool — making every tool a thin client that relies on Yuva AI's agent system via CLI commands. Native configs act as routing layers, not standalone instruction sets.

## Problem

Currently `yuva init` creates a pointer file ("go read AGENTS.md") for non-default tools. AI tools don't leverage their native features (slash commands, scoped rules, settings). Users get a degraded experience compared to tools that read AGENTS.md directly.

## Goals

1. Every supported AI tool gets a native config that uses its unique features
2. All native configs enforce dependency on `yuva` CLI — no embedded prompts
3. Agent prompts remain single source of truth in `.aiautomations/prompts/`
4. Generated configs are gitignored — only `AGENTS.md` and `.aiautomations/` are committed
5. `yuva init --all` generates configs for every tool at once

## Non-Goals

- Embedding full agent prompts in native configs
- Replacing `.aiautomations/` with tool-specific configs
- Supporting tools not already in `LLM_CONFIGS`

---

## Command Interface

```
yuva init                     # interactive: auto-detect → confirm → generate
yuva init --all               # generate native configs for ALL tools
yuva init --tool <name>       # generate for specific tool (no prompt)
yuva init --force             # overwrite existing configs
yuva init --dry-run           # preview what would be created
```

## Core Principle: Native Configs as Thin Drivers

Every generated native config follows this template:

```markdown
# Yuva AI - [Tool Name] Configuration
<!-- Auto-generated by yuva init. Do not edit — regenerate with: yuva init --tool <name> --force -->

## CRITICAL: Before ANY task
You MUST run `yuva agent orchestrate` first. Do NOT start work without it.
Read the JSON output to understand the project state.

## How to load agent instructions
NEVER guess or improvise agent behavior.
Run: `yuva agent show <name>` to get the EXACT prompt for any agent.
Follow each agent's instructions completely before moving to the next.

## Agent Selection Guide

| User wants to...        | Run these agents in order                                          |
|-------------------------|--------------------------------------------------------------------|
| Build something new     | `yuva agent show requirements` → `riskassessment` → `planning` → `execution` |
| Continue previous work  | `yuva agent show continuity` → (resume from last point)           |
| Fix a bug / error       | `yuva agent show debugger`                                         |
| Write tests             | `yuva agent show tester`                                           |
| Review code quality     | `yuva agent show reviewer`                                        |
| Check security          | `yuva agent show security`                                         |
| Clean up / improve code | `yuva agent show refactor`                                         |

## Rules
1. Always run `yuva agent orchestrate` first
2. If existing code exists, run `yuva agent show existingcode` BEFORE any other agent
3. One agent at a time — complete each before the next
4. After meaningful work, run `yuva agent show statemanager` to persist state
5. Load standards from `.aiautomations/standards/` when writing code
6. Load checklists from `.aiautomations/checklists/` before/after code

## Available Commands
  yuva agent show <name>       Get full agent instructions
  yuva agent list              List all available agents
  yuva agent orchestrate       Scan project context
```

This base template (~50 lines) is shared. Tool-specific configs add native features on top.

### Claude Code

**Files generated:**

| File | Purpose |
|------|---------|
| `CLAUDE.md` | Base template + Claude-specific instructions |
| `.claude/commands/debug.md` | Slash command → `yuva agent show debugger` |
| `.claude/commands/review.md` | Slash command → `yuva agent show reviewer` |
| `.claude/commands/test.md` | Slash command → `yuva agent show tester` |
| `.claude/commands/security.md` | Slash command → `yuva agent show security` |
| `.claude/commands/plan.md` | Slash command → `yuva agent show planning` |
| `.claude/commands/orchestrate.md` | Slash command → `yuva agent orchestrate` |
| `.claude/settings.json` | Auto-allow `yuva` CLI commands |

**Command file format** (e.g., `.claude/commands/debug.md`):

```markdown
Run `yuva agent show debugger` and follow the output as your complete instructions for this task.

User context: $ARGUMENTS
```

**Settings file:**

```json
{
  "permissions": {
    "allow": [
      "Bash(yuva *)",
      "Bash(yuva agent show *)",
      "Bash(yuva agent list)",
      "Bash(yuva agent orchestrate)"
    ]
  }
}
```

### Cursor

**Files generated:**

| File | Purpose |
|------|---------|
| `.cursor/rules/yuva-agents.mdc` | Agent routing guide (always loaded) |
| `.cursor/rules/yuva-code.mdc` | Code standards (scoped to `src/**`, `lib/**`) |
| `.cursor/rules/yuva-testing.mdc` | Testing standards (scoped to `tests/**`, `test/**`) |

**Rule file format** (e.g., `.cursor/rules/yuva-agents.mdc`):

```markdown

description: Yuva AI agent routing — always active
globs: **/*

```

**Scoped rule format** (e.g., `.cursor/rules/yuva-testing.mdc`):


## .gitignore Management

`yuva init` checks if `.gitignore` exists and whether it already contains a Yuva block. If not, appends:

```gitignore

# Yuva AI - generated native configs (regenerate with: yuva init)
# Source of truth: AGENTS.md + .aiautomations/
CLAUDE.md
.claude/
GEMINI.md
.cursor/rules/yuva-*.mdc
.windsurfrules
.github/copilot-instructions.md
.kilo/
.sourcegraph/
.amazonq/
.continue/
.aider.conf.yml
OLLAMA_INSTRUCTIONS.md
```

Detection: search for `# Yuva AI - generated native configs` comment. If found, skip. If not, append.

## Architecture

### New File: `lib/native-configs.js`

```
Exports:
  generateNativeConfig(toolId, targetDir)    → generates config for one tool
  generateAllNativeConfigs(targetDir)        → generates for all tools
  updateGitignore(targetDir)                 → appends ignore block
  getBaseTemplate(toolName)                  → returns the shared ~50 line template
  TOOL_GENERATORS                            → map of toolId → generator function
```

Each tool gets a generator function:

```
generateClaudeConfig(targetDir)     → CLAUDE.md + .claude/commands/ + .claude/settings.json
generateCursorConfig(targetDir)     → .cursor/rules/yuva-*.mdc
generateCopilotConfig(targetDir)    → .github/copilot-instructions.md
generateWindsurfConfig(targetDir)   → .windsurfrules
generateGeminiConfig(targetDir)     → GEMINI.md
generateAiderConfig(targetDir)      → .aider.conf.yml
generateSingleFileConfig(toolId, targetDir) → generic handler for single-file tools
generateOllamaConfig(targetDir)     → OLLAMA_INSTRUCTIONS.md
```

### Modified: `lib/commands/init.js`

Changes:
1. Add `--all` flag parsing
2. Add interactive prompt: auto-detect → confirm → fallback to list
3. After creating `AGENTS.md` + `.aiautomations/`, call `generateNativeConfig()` or `generateAllNativeConfigs()`
4. Call `updateGitignore()`
5. Updated success output showing all generated files

### Interactive Prompting

Use Node.js `readline` (no external dependencies) for the confirm/select prompts.

## Dependency Chain

```
.aiautomations/prompts/  ← single source of truth (agent prompts live here)
        ↑
    yuva CLI             ← serves prompts via `yuva agent show`
        ↑
  Native configs         ← thin drivers that call yuva CLI
        ↑
    AI Tool              ← reads native config, executes yuva commands
```

Updating a prompt in `.aiautomations/prompts/` instantly changes behavior for ALL tools — no regeneration needed. Native configs only need regeneration if the routing/structure changes (new agents, new commands).

---

## Edge Cases

1. **No .gitignore exists** — create one with the Yuva block
2. **Yuva block already in .gitignore** — skip, don't duplicate
3. **User has existing .cursor/rules/** — only write `yuva-*.mdc` files, don't touch others
4. **User has existing .github/copilot-instructions.md** — overwrite only with `--force`
5. **Tool not detected** — fall back to interactive list (no default assumption)
6. **`yuva` CLI not installed globally** — native configs reference `npx yuva-ai` as fallback
7. **`--dry-run` with `--all`** — list ALL files that would be created across all tools
